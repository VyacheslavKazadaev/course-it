## 📅 НЕДЕЛЯ 1: JavaScript ES6+ и DOM

*Цель:* Освоить современный синтаксис JS, понять работу DOM, научиться манипулировать элементами и обрабатывать события.

### День 1: Переменные, функции, шаблонные строки (ES6+)

📖 **Теория & Инструкция**

- `let`/`const` вместо `var` (блочная область видимости)
- Стрелочные функции, параметры по умолчанию
- Template literals: `` `Привет, ${name}!` ``
- Отладка: `console.table()`, точки останова в DevTools → Sources

💻 **Интерактивное задание в браузере**

1. Откройте [JSFiddle](https://jsfiddle.net/) или [CodePen](https://codepen.io/)
2. Вставьте стартовый код:

```html
<div id="profile"></div>
```

```js
const user = { name: "Анна", role: "Junior", exp: 1 };
// Напишите функцию renderProfile(user), которая вернет строку:
// "👤 Анна | Роль: Junior | Опыт: 1 г."
// Вставьте результат в #profile через textContent
```

3. Замените `function` на стрелочную, используйте template literals.

📝 **ДЗ:** Создать генератор карточки товара (название, цена, скидка, итоговая цена). Вывести в DOM.
✅ **Критерии:** Нет `var`, используются `const`/`let`, код читается слева направо, нет ошибок в консоли.
🔍 **Совет:** Откройте Console → вкладка Sources → поставьте breakpoint на `renderProfile` → пошагово выполните код.

### День 2: Массивы и объекты: деструктуризация, spread/rest, методы

📖 **Теория & Инструкция**

- `map`, `filter`, `reduce` (цепочки, иммутабельность)
- Деструктуризация объектов/массивов
- Spread `...arr`, Rest `...args`
- Опциональная цепочка `?.`

💻 **Интерактивное задание**

1. Откройте [Replit JS Template](https://replit.com/new/js)
2. Дан массив: `const products = [{id:1, name:"Ноутбук", price:50000, inStock:true}, ...]`
3. Напишите:
   - Отфильтровать доступные товары
   - Преобразовать в массив строк: `"Ноутбук — 50 000 ₽"`
   - Посчитать общую сумму через `reduce`
4. Выведите результаты через `console.table`

📝 **ДЗ:** Реализовать корзину: `addToCart`, `removeFromCart`, `getTotal`. Без мутации исходного массива.
✅ **Критерии:** Чистые функции, нет `for`-циклов там, где есть методы массивов, используется деструктуризация.
🔍 **Совет:** `reduce` часто путают со `map`. Помните: `reduce` → одно значение, `map` → массив той же длины.

## 📅 НЕДЕЛЯ 2: Асинхронность и работа с сетью

*Цель:* Понять Event Loop, освоить Promise/async-await, научиться запрашивать данные и обрабатывать ошибки.

### День 6: Callbacks → Promises → async/await

📖 **Теория & Инструкция**

- Почему колбэки ведут к "hell"
- Promise: `new Promise`, `resolve/reject`, `.then/.catch`
- `async/await` синтаксис, параллельное выполнение `Promise.all`

💻 **Интерактив:** [Promise Visualizer](https://jakearchibald.com/2014/things-to-know-about-js/#promises) + [CodeSandbox JS](https://codesandbox.io/s/vanilla)
Задание: Симулировать загрузку данных с задержкой 1с, 2с, 3с. Вывести по порядку через `async/await`. Затем через `Promise.all`.

📝 **ДЗ:** Написать функцию `retry(fn, attempts, delay)`, которая повторяет промис при ошибке.
✅ **Критерии:** Нет вложенных `.then`, используется `try/catch`, `Promise.all` для независимых запросов.
🔍 **Совет:** `await` работает только внутри `async` функции. Ошибка в `await` прерывает выполнение → оборачивайте в `try/catch`.

📖 **Теория & Инструкция**

💻 **Интерактив:** [JSONPlaceholder](https://jsonplaceholder.typicode.com/users)

📝 **ДЗ:** Реализовать поиск по пользователям через `fetch` + debouncing (задержка ввода 500мс).
✅ **Критерии:** Проверка `response.ok`, обработка network errors, UI не зависает при медленном интернете.
🔍 **Совет:** `fetch` не реджектит на 4xx/5xx. Всегда: `if (!res.ok) throw new Error(res.status)`.
