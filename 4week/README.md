## 📚 ДЕНЬ 1: Введение в Linux

### 🎯 Теория

**Что такое Linux?**

- Linux — это операционная система с открытым исходным кодом
- Ядро (Kernel) + пользовательские утилиты (GNU)
- Существует в виде дистрибутивов: Ubuntu, Debian, CentOS, Fedora

**Файловая система Linux:**

```
/                    # Корневая директория
├── bin/             # Бинарные файлы (команды)
├── home/            # Домашние директории пользователей
├── etc/             # Конфигурационные файлы
├── var/             # Переменные данные (логи)
├── tmp/             # Временные файлы
├── usr/             # Пользовательские программы
└── root/            # Домашняя директория root
```

## 📚 ДЕНЬ 4: Права доступа и пользователи

### 🎯 Теория

**Система прав в Linux:**

```
-rwxrwxrwx  1  user  group  4096  Jan 1 12:00  file.txt
││││││││││
││││││└─────── Прочие (others)
│││└─────────── Группа (group)
││└──────────── Владелец (user)
│└───────────── Тип (файл/директория)
└────────────── Права: r=read, w=write, x=execute
```

**Команды управления правами:**

| Команда | Описание                             | Пример              |
| -------------- | -------------------------------------------- | ------------------------- |
| `chmod`      | Изменить права                  | `chmod 755 file.sh`     |
| `chown`      | Изменить владельца          | `chown user:group file` |
| `sudo`       | Выполнить от root                 | `sudo apt update`       |
| `id`         | Показать ID пользователя | `id`                    |

**Числовые значения прав:**

- r (read) = 4
- w (write) = 2
- x (execute) = 1

### 🖥️ Интерактивное задание #4

**Задание: Управление правами**

```bash
# 1. Создайте файл и скрипт
cd ~/linux_practice
touch secret.txt
echo '#!/bin/bash' > script.sh
echo 'echo "Hello from script"' >> script.sh

# 2. Посмотрите текущие права
ls -l secret.txt script.sh

# 3. Измените права на скрипт (сделайте исполняемым)
chmod 755 script.sh

# 4. Измените права на файл (только чтение для всех)
chmod 644 secret.txt

# 5. Проверьте изменения
ls -l

# 6. Попробуйте выполнить скрипт
./script.sh
```

**📝 Вопрос для размышления:** Почему `chmod 755` делает файл исполняемым? Запишите ответ.

<details>
<summary>✅ Ответ</summary>

755 = 7 (владелец: rwx=4+2+1) + 5 (группа: r-x=4+0+1) + 5 (остальные: r-x=4+0+1)
Бит выполнения (x=1) установлен для всех категорий пользователей.

</details>

## 📚 ДЕНЬ 5: Текстовые редакторы

### 🎯 Теория

**Два основных редактора:**

| Редактор | Сложность | Использование                    |
| ---------------- | ------------------ | --------------------------------------------- |
| `nano`         | Легкий       | Быстрое редактирование   |
| `vim`          | Средний     | Профессиональная работа |

**nano — базовые команды:**

- `Ctrl+O` — сохранить
- `Ctrl+X` — выйти
- `Ctrl+K` — вырезать строку
- `Ctrl+U` — вставить строку

**vim — режимы:**

- `i` — режим вставки (Insert)
- `Esc` — нормальный режим
- `:w` — сохранить
- `:q` — выйти
- `:wq` — сохранить и выйти

## 📚 ДЕНЬ 6: Процессы и системные команды

### 🎯 Теория

**Команды мониторинга:**

| Команда | Описание                                        | Пример             |
| -------------- | ------------------------------------------------------- | ------------------------ |
| `ps`         | Показать процессы                       | `ps aux`               |
| `top`        | Мониторинг в реальном времени | `top`                  |
| `kill`       | Завершить процесс                       | `kill PID`             |
| `grep`       | Поиск в тексте                              | `grep "error" log.txt` |
| `pipe \|`     | Передать вывод другой команде | `ps aux \| grep python` |
