## 🟦 НЕДЕЛЯ 1: Синтаксис Go, Экосистема, Python vs Go

### 📅 День 1: Введение в Go, типы, пакеты, модули

- 🎯 **Цель:** Написать первую программу, понять систему типов и `go mod`
- 📖 **Теория:** `main`, `fmt`, статическая типизация, zero-values, `go run/build`, модули
- 🛠 **Инструкция:**
  1. Создайте `go mod init month5-go`
  2. Напишите программу, читающую ввод, парсящую числа, выводящую сумму
  3. Разбейте на пакеты `input/` и `calc/`
- 💻 **Интерактив:** [Go Playground](https://go.dev/play/) → задача: реализовать функцию `SumSlice([]int) int` с обработкой пустого среза
- ✅ **Самопроверка:** `go vet`, `go fmt`, `go run` без ошибок, `gopls` подсвечивает типы
- 🔗 **Проект:** Подготовка структуры для будущего API-переписывания

### 📅 День 2: Структуры, методы, интерфейсы, ошибки

- 🎯 **Цель:** Понять композицию вместо наследования, контрактные интерфейсы
- 📖 **Теория:** `struct`, embedding, методы с value/receiver, `error`, `fmt.Errorf`, `errors.Is/As`
- 🛠 **Инструкция:** Создайте `User` с методом `Validate() error`. Реализуйте интерфейс `Notifier` (email/sms)
- 💻 **Интерактив:** [Replit Go Template](https://replit.com/@replit/Go) → добавьте 3 реализации `Notifier`, протестируйте через `switch.(type)`
- ✅ **Самопроверка:** `go test -v ./...`, покрытие >70%
- 🔗 **Проект:** Модели данных для Python→Go миграции

### 📅 День 3: Python vs Go: когда что выбирать

- 🎯 **Цель:** Научиться аргументировать выбор стека под задачу
- 📖 **Теория:**
  - Python: быстрый прототип, DS/ML, экосистема библиотек, GIL
  - Go: высоконагруженные сервисы, конкурентность, низкий footprint, строгая типизация, компиляция
  - Бенчмарки: HTTP RPS, память, время запуска
- 🛠 **Инструкция:** Составьте таблицу сравнения по 5 параметрам. Напишите мини-обоснование для 3 кейсов (ML-пайплайн, грейплайн платежей, real-time нотификации)
- 💻 **Интерактив:** [Go vs Python Benchmark Playground](https://replit.com/@community/go-vs-python-bench) → запустите, скопируйте метрики, ответьте на вопросы в чек-листе
- ✅ **Самопроверка:** Ответы совпадают с эталоном (Go: ~5-10x RPS, <50MB RAM; Python: быстрее dev-цикл, богатые фреймворки)
- 🔗 **Проект:** Выбор эндпоинтов для переписывания (наиболее CPU/IO-bound)

### 📅 День 4: HTTP-сервер в Go, маршрутизация, middleware

- 🎯 **Цель:** Поднять REST-сервер без фреймворков, добавить логирование и восстановление от паник
- 📖 **Теория:** `net/http`, `http.HandlerFunc`, `context`, middleware chain
- 🛠 **Инструкция:** Напишите сервер с `/health`, `/echo`, `/slow`. Добавьте middleware: logging, recovery, request-id
- 💻 **Интерактив:** [Go HTTP Sandbox](https://replit.com/@replit/Go-HTTP-Server) → вызовите endpoints через встроенный Preview, проверьте заголовки
- ✅ **Самопроверка:** `curl -v localhost:8080/slow` → таймаут 2с, лог с request-id
- 🔗 **Проект:** Каркас для переписанного Python API

### 📅 День 5: JSON, валидация, тестирование HTTP

- 🎯 **Цель:** Парсить/сериализовать JSON, писать интеграционные тесты
- 📖 **Теория:** `encoding/json`, теги struct, `httptest.NewRecorder`, table-driven tests
- 🛠 **Инструкция:** Добавьте эндпоинт `POST /users` с валидацией email. Напишите 3 теста: успех, ошибка валидации, дубликат
- 💻 **Интерактив:** [Go Test Runner (browser)](https://go.dev/play/p/testrunner_sim) → вставьте код, запустите, исправьте failing cases
- ✅ **Самопроверка:** `go test -cover` → ≥80%, `go vet` clean
- 🔗 **Проект:** Готовый модуль `/users` для миграции с Python

## 🟩 НЕДЕЛЯ 2: Конкурентность + Базы Данных

### 📅 День 6: Горутины: планировщик, `go`, утечки

- 🎯 **Цель:** Понять M:N модель, запускать горутин, избегать goroutine leaks
- 📖 **Теория:** `go func()`, scheduler, `runtime.GOMAXPROCS`, отладка `runtime.NumGoroutine()`
- 🛠 **Инструкция:** Напишите функцию, обрабатывающую 100 задач конкурентно. Добавьте `time.Sleep` и лог старта/окончания
- 💻 **Интерактив:** [Goroutine Visualizer (web)](https://replit.com/@community/goroutine-viz) → запустите, наблюдайте за счетчиком горутин, найдите leak (незакрытый канал)
- ✅ **Самопроверка:** Количество горутин возвращается к 1 после завершения
- 🔗 **Проект:** Подготовка к конкурентной обработке запросов в API

### 📅 День 7: Каналы: unbuffered, buffered, `select`, `close`

- 🎯 **Цель:** Освоить коммуникацию между горутинами
- 📖 **Теория:** направление каналов, блокировки, fan-in/fan-out, graceful shutdown через закрытие канала
- 🛠 **Инструкция:** Реализуйте worker pool с buffered channel. Добавьте `select` с `context.Done()` для отмены
- 💻 **Интерактив:** [Go Playground: Channels](https://go.dev/play/p/channel_workshop) → выполните 3 задачи: timeout, retry, cancellation
- ✅ **Самопроверка:** Тесты проходят, `race detector` (`go test -race`) чист
- 🔗 **Проект:** Паттерн для обработки входящих HTTP-запросов в пуле

### 📅 День 8: Синхронизация: WaitGroup, Mutex, Atomic

- 🎯 **Цель:** Выбирать правильный примитив под задачу
- 📖 **Теория:** `sync.WaitGroup`, `sync.RWMutex`, `sync/atomic`, когда не использовать каналы
- 🛠 **Инструкция:** Напишите thread-safe counter с Mutex и Atomic. Сравните производительность
- 💻 **Интерактив:** [Race Condition Simulator](https://replit.com/@community/go-race-sim) → запустите с/без `-race`, зафиксируйте разницу
- ✅ **Самопроверка:** `go run -race` не падает, бенчмарк показывает ~2x скорость atomic
- 🔗 **Проект:** Безопасное кеширование счетчиков в аналитике

### 📅 День 9: Оптимизация SQL-запросов

- 🎯 **Цель:** Читать EXPLAIN, выбирать индексы, избегать N+1
- 📖 **Теория:** B-Tree vs Hash индексы, составные индексы, `EXPLAIN ANALYZE`, connection pool (`database/sql`)
- 🛠 **Инструкция:** Создайте таблицу `events`. Напишите запрос с `WHERE`, `ORDER BY`, `JOIN`. Добавьте индекс, сравните планы
- 💻 **Интерактив:** [DB Fiddle](https://www.db-fiddle.com/) → вставьте схему, запустите EXPLAIN, оптимизируйте до `Using index`
- ✅ **Самопроверка:** Время выполнения <50ms, план использует index scan
- 🔗 **Проект:** Оптимизация запросов для переписанного API

### 📅 День 10: Миграции БД в Go

- 🎯 **Цель:** Управлять версиями схемы, откатывать изменения
- 📖 **Теория:** `golang-migrate`, up/down файлы, транзакции, idempotency
- 🛠 **Инструкция:** Инициализируйте `migrate`. Создайте `001_users.up.sql`/`.down.sql`. Примените, откатите, проверьте состояние
- 💻 **Интерактив:** [Migrate CLI Simulator](https://replit.com/@community/go-migrate-cli) → выполните команды, ответьте на вопросы по состоянию БД
- ✅ **Самопроверка:** `migrate version` показывает корректную версию, rollback восстанавливает схему
- 🔗 **Проект:** Добавление миграций в репозиторий Python→Go API

### 📅 День 13: Проект 1. Переписывание Python API на Go (часть 1)

1. **Интерактив в браузере:** Все ссылки ведут на стабильные бесплатные платформы. Если доступ ограничен, используйте локальные Docker-контейнеры + VS Code Dev Containers.

   - Dockerized деплой + CI

   - 60%: корректность кода, тесты, обработка ошибок

   - 20%: документация, архитектура, готовность к продакшену
