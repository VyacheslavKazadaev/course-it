### 📘 Месяц 8: Подготовка к трудоустройству + Продвинутые темы

**Цель месяца:** Сформировать готовое портфолио, пройти симуляцию собеседований, развернуть финальный проект в облаке и подготовиться к активному поиску работы.
**Стек:** Backend на `Go` или `Python` (в заданиях указаны адаптации для обоих языков).
**Формат:** 4 недели, 7 дней в неделю. Каждый день содержит текстовую инструкцию, интерактивное задание в браузере и практику. Все инструменты работают без установки ПО.

## 🗓 НЕДЕЛЯ 1: Алгоритмы + GitHub/Портфолио + Open-Source

### 📅 Понедельник | Алгоритмы: сложность, сортировки, бинарный поиск

📖 **Теория:** Разберём Big-O, когда использовать `O(n log n)` vs `O(n²)`, принципы бинарного поиска и стабилизации сортировок.💻 **Интерактив в браузере:**

1. Откройте [VisuAlgo](https://visualgo.net/en/sorting) → запустите `QuickSort` и `MergeSort` с массивом 100 элементов. Запишите количество сравнений.
2. Перейдите в [LeetCode](https://leetcode.com) → задача `704. Binary Search`. Используйте онлайн-редактор, выберите язык (Go/Python), напишите решение, нажмите `Run` → `Submit`.
   🛠 **Практика:** Решите 2 задачи уровня Easy на массивах. Сохраните решения в репозитории `algorithms-practice`.
   ✅ **Итог:** Понимаете, как оценивать сложность и выбирать алгоритм под задачу.

### 📅 Вторник | GitHub профиль: структура, README, CI-бейджи

📖 **Теория:** Профиль разработчика = ваша визитка. Важны: `README.md`, закреплённые репозитории, Actions, статусы, чистая история коммитов.💻 **Интерактив в браузере:**

1. Откройте `github.com/<ваш-ник>?tab=repositories`. Нажмите `+` → `New repository` → `Initialize with README`.
2. Перейдите в [GitHub Profile README Generator](https://rahuldkjain.github.io/gh-profile-readme-generator/) → сгенерируйте шаблон, скопируйте код.
3. В GitHub откройте `README.md` → нажмите `🖊️ Edit` → вставьте шаблон → `Commit`.
   🛠 **Практика:** Добавьте секции: `Tech Stack`, `Current Focus`, `Stats` (используйте `github-readme-stats`), `Contact`.
   ✅ **Итог:** Профиль выглядит профессионально и отражает навыки.

### 📅 Среда | Open-Source: поиск задач, форк, первый PR

📖 **Теория:** Контрибьютинг начинается с `good first issue`. Важно: читать `CONTRIBUTING.md`, следовать стилю, писать понятные коммиты.💻 **Интерактив в браузере:**

1. Откройте [GitHub Explore: Good First Issues](https://github.com/topics/good-first-issue). Отфильтруйте по языку (`go` или `python`).
2. Выберите проект → `Fork` → `Code` → `Open with Codespaces` (браузерная IDE).
3. Внесите мелкое правку (исправьте опечатку в документации или добавьте комментарий).
   🛠 **Практика:** Создайте ветку `fix/doc-typo`, сделайте коммит, отправьте PR. Опишите изменения по шаблону.
   ✅ **Итог:** Понимаете цикл open-source контрибьюшена.

### 📅 Четверг | Системный дизайн: основы, CAP, балансировка, кэш

📖 **Теория:** CAP-теорема, горизонтальное vs вертикальное масштабирование, роль LB и кэша, stateless-архитектура.💻 **Интерактив в браузере:**

1. Откройте [Excalidraw](https://excalidraw.com). Нарисуйте схему: `Client → CDN → LB → Web App → Cache → DB`.
2. Перейдите в [ByteByteGo Free Newsletter](https://blog.bytebytego.com) → прочитайте раздел про Load Balancers.
   🛠 **Практика:** Спроектируйте URL-шортнер. Опишите потоки данных, выберите БД (PostgreSQL/Redis), укажите точки отказа.
   ✅ **Итог:** Умеете декомпозировать систему на компоненты.

### 📅 Пятница | Алгоритмы: хэш-таблицы, строки, два указателя

📖 **Теория:** Когда использовать `map/dict` для `O(1)` поиска, техника двух указателей для оптимизации проходов по массиву/строке.💻 **Интерактив в браузере:**

1. Откройте [NeetCode.io](https://neetcode.io) → раздел `Two Pointers` → задача `125. Valid Palindrome`.
2. В онлайн-редакторе напишите решение на Go (`strings.ToLower`, `[]rune`) или Python (`s.lower()`, `s.replace()`).
   🛠 **Практика:** Решите `3. Longest Substring Without Repeating Characters`. Добавьте поясняющие комментарии.
   ✅ **Итог:** Закреплены паттерны для кодинговых собеседований.

### 📅 Суббота | Портфолио: GitHub Pages + демо проектов

📖 **Теория:** Рабочие демо важнее кода. Статичный сайт с ссылками на репозитории, скриншотами и live-превью.💻 **Интерактив в браузере:**

1. Создайте репозиторий `<ник>.github.io`.
2. В `index.html` добавьте ссылки на 3 проекта, описание стека, кнопку `View Demo`.
3. В настройках репозитория → `Pages` → `Source: GitHub Actions`.
   🛠 **Практика:** Настройте деплой, проверьте доступность по `https://<ник>.github.io`.
   ✅ **Итог:** Портфолио доступно по ссылке, работает на любом устройстве.

### 📅 Воскресенье | Ретроспектива + план Capstone

📖 **Теория:** Capstone должен решать реальную проблему, иметь API, БД, авторизацию, тесты, CI/CD.💻 **Интерактив в браузере:**

1. Откройте [Notion](https://notion.so) или [Trello](https://trello.com). Создайте доску: `Backlog → In Progress → Review → Done`.
2. Добавьте задачи: `API scaffold`, `Auth`, `CRUD`, `Tests`, `Deploy`, `Docs`.
   🛠 **Практика:** Выберите тему (например: `Task Manager API`, `URL Shortener`, `Microblog Backend`). Напишите `README.md` с описанием, стеком, архитектурой.
   ✅ **Итог:** Чёткий план финального проекта готов.

## 🗓 НЕДЕЛЯ 2: Системный дизайн + Сеть/Облака + Блог/Документация

### 📅 Понедельник | Системный дизайн: БД, индексы, пагинация, очереди

📖 **Теория:** Когда использовать SQL vs NoSQL, B-Tree индексы, курсорная vs offset пагинация, RabbitMQ/Redis Queue.💻 **Интерактив в браузере:**

1. Откройте [DB Designer](https://dbdesigner.net). Создайте схему `Users → Posts → Comments` с FK и индексами.
2. Пройдите [SQLZoo](https://sqlzoo.net) → раздел `SELECT basics`.
   🛠 **Практика:** Напишите SQL-запросы для пагинации (`LIMIT/OFFSET` → курсорная). Добавьте в `design/` Capstone.
   ✅ **Итог:** Понимаете проектирование схем и оптимизацию запросов.

### 📅 Вторник | Сетевые технологии: HTTP/HTTPS, REST, gRPC, WebSocket

📖 **Теория:** Структура запроса, статус-коды, идемпотентность, отличие REST от gRPC (protobuf), когда нужен WS.💻 **Интерактив в браузере:**

1. Откройте [Postman Web](https://web.postman.co). Создайте коллекцию `Capstone API`.
2. Отправьте `GET/POST/PUT/DELETE` на `https://httpbin.org`. Зафиксируйте заголовки, статусы, тело.
   🛠 **Практика:** Реализуйте маршруты Capstone. Для Python: `FastAPI`; для Go: `gin` или `echo`. Добавьте валидацию.
   ✅ **Итог:** API соответствует стандартам REST.

### 📅 Среда | Облачные сервисы: VM, S3, IAM, Docker в браузере

📖 **Теория:** Облачные провайдеры, объектные хранилища, роли и политики, контейнеризация.💻 **Интерактив в браузере:**

1. Откройте [Play with Docker](https://labs.play-with-docker.com). Создайте инстанс.
2. Запустите: `docker run -d -p 5432:5432 postgres:15-alpine`. Подключитесь через `docker exec`.
   🛠 **Практика:** Напишите `Dockerfile` для Capstone (Python: `python:3.11-slim`, Go: `golang:1.22-alpine`). Соберите и запустите локально через `docker compose`.
   ✅ **Итог:** Приложение контейнеризовано, готово к облачному деплою.

### 📅 Четверг | Блог/Документация: MkDocs/Hugo + GitHub Pages

📖 **Теория:** Технический блог показывает глубину понимания. Автоматизация сборки через CI.💻 **Интерактив в браузере:**

1. Откройте [GitHub Codespaces](https://github.com/features/codespaces) → создайте из шаблона `mkdocs-material` или `hugo`.
2. В `docs/index.md` напишите статью `Как я проектировал API для Capstone`.
   🛠 **Практика:** Настройте `.github/workflows/deploy.yml` → авто-деплой в GitHub Pages. Опубликуйте 2 статьи.
   ✅ **Итог:** Блог работает, документация проекта обновлена.

### 📅 Пятница | Сетевая безопасность: CORS, JWT, OAuth2, Rate Limiting

📖 **Теория:** Как браузер проверяет источники, структура токена, поток авторизации, защита от брутфорса.💻 **Интерактив в браузере:**

1. Откройте [jwt.io](https://jwt.io). Вставьте тестовый токен, измените payload, посмотрите, как ломается подпись.
2. Перейдите в [OAuth 2.0 Playground](https://developers.google.com/oauthplayground). Пройдите поток `Authorization Code`.
   🛠 **Практика:** Добавьте JWT-аутентификацию в Capstone. Для Python: `python-jose` + `fastapi-security`; для Go: `golang-jwt`.
   ✅ **Итог:** API защищён, токены валидируются, CORS настроен.

### 📅 Суббота | CI/CD в облаке: GitHub Actions + автоматизация

📖 **Теория:** Пайплайн: `lint → test → build → push → deploy`. Секреты, кеширование, ручные/автоматические триггеры.💻 **Интерактив в браузере:**

1. В репозитории Capstone → `Actions` → `New workflow`. Выберите `Go` или `Python`.
2. Добавьте шаги: `checkout`, `setup`, `lint`, `test`, `docker build`, `deploy to Render/Vercel`.
   🛠 **Практика:** Настройте пайплайн. Протестируйте на `push` и `PR`.
   ✅ **Итог:** Деплой автоматизирован, ошибки отлавливаются до мержа.

### 📅 Воскресенье | Продвинутый системный дизайн: шардирование, репликация, отказоустойчивость

📖 **Теория:** Когда делить БД, как работает leader/follower, circuit breaker, health checks.💻 **Интерактив в браузере:**

1. Откройте [Excalidraw](https://excalidraw.com). Нарисуйте архитектуру с `Read Replica`, `Connection Pooler`, `Health Endpoint`.
2. Прочитайте раздел `Scalability` в [System Design Primer](https://github.com/donnemartin/system-design-primer).
   🛠 **Практика:** Перепишите раздел `Architecture` в Capstone README. Добавьте схему отказоустойчивости.
   ✅ **Итог:** Проект спроектирован с учётом масштабирования.
